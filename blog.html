Frequently Asked Questions

Q. Discuss the scope of var, let, and const.

A. var has function scope and allows redeclaration. It's hoisted, which can lead to issues.
let has block scope, doesn't allow redeclaration, and isn't hoisted. It's preferred for mutable variables.
const also has block scope, doesn't allow reassignment (for primitives), and isn't hoisted. It's preferred for immutable variables.

Q. Tell us the use cases of null and undefined.

A. undefined is used to indicate the absence of a value or uninitialized variables.
It's automatically assigned to variables with no value, function parameters without arguments, or when a function doesn't explicitly return a value.
null is used to explicitly indicate the absence of a value or empty objects.
It's typically assigned to variables or properties to represent intentional absence or emptiness.

Q. What do you mean by REST API?.

A. A REST API is an architectural approach for designing web services that use standard HTTP methods like GET (retrieve), POST (create), PUT (update), and DELETE (delete) to interact with resources identified by unique URLs. It's stateless, allowing for scalability, and supports various data representations like JSON or XML. It treats everything as a resource and organizes them using URLs.
